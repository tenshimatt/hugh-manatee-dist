# Rawgle Backend Deployment Guide

Complete guide for deploying the Rawgle backend API to Cloudflare Workers.

## 🚀 Pre-Deployment Checklist

### 1. Existing Database Setup
- ✅ **Existing Rawgle D1 Database**: Use the production database with 9000+ supplier records
- ✅ **Database ID**: Update `wrangler.toml` with the correct `database_id`
- ✅ **Schema Compatibility**: The backend is designed to work with existing schema

### 2. Required Cloudflare Resources

Run the setup script to create necessary resources:
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

This creates:
- **KV Namespace**: For sessions and caching
- **R2 Bucket**: For media storage (photos, videos)
- **Queue**: For background processing
- **Preview environments**: For development and staging

### 3. Environment Configuration

Update `wrangler.toml` with your resource IDs:

```toml
# D1 Database - Use existing Rawgle database
[[d1_databases]]
binding = "DB"
database_name = "rawgle-production"
database_id = "YOUR_EXISTING_RAWGLE_DB_ID"  # ← Update this

# KV Namespace - Generated by setup script
[[kv_namespaces]]
binding = "KV"
id = "YOUR_KV_NAMESPACE_ID"  # ← Generated by setup

# R2 Bucket - Created by setup script
[[r2_buckets]]
binding = "R2"
bucket_name = "rawgle-media"
```

## 🔧 Configuration Steps

### 1. Update Frontend URL
Ensure CORS is configured for your frontend:

```toml
[vars]
RAWGLE_FRONTEND_URL = "https://afc39a6e.rawgle-frontend.pages.dev"
CORS_ORIGINS = ["https://afc39a6e.rawgle-frontend.pages.dev", "http://localhost:3000"]
```

### 2. Set JWT Secret
⚠️ **Important**: Change the default JWT secret for production:

```javascript
// In src/middleware/auth.js and src/services/auth-service.js
const jwtSecret = c.env.JWT_SECRET || 'your-production-secret-here';
```

Add to `wrangler.toml`:
```toml
[vars]
JWT_SECRET = "your-secure-jwt-secret-key-here"
```

### 3. Database Migration (If Needed)
If the existing database needs updates:

```bash
# Check current schema
wrangler d1 execute rawgle-production --command "SELECT name FROM sqlite_master WHERE type='table';"

# Apply migrations if needed
wrangler d1 migrations apply rawgle-production
```

## 🔒 Security Considerations

### 1. JWT Secret
- Use a strong, unique JWT secret for production
- Rotate JWT secrets periodically
- Store secrets securely in Cloudflare environment variables

### 2. Rate Limiting
The backend includes built-in rate limiting:
- Authentication endpoints: 5 attempts per 15 minutes
- File uploads: 20 uploads per 5 minutes
- PAWS transfers: 10 transfers per hour

### 3. CORS Configuration
Ensure CORS is properly configured for your frontend domain only.

### 4. Input Validation
All endpoints include comprehensive input validation using Zod schemas.

## 🆘 Troubleshooting

### Common Issues

**1. Database Connection Errors**
```bash
# Check database binding
wrangler d1 list

# Test database connection
wrangler d1 execute rawgle-production --command "SELECT COUNT(*) FROM suppliers;"
```

**2. CORS Errors**
- Verify frontend URL is in `CORS_ORIGINS`
- Check that preflight OPTIONS requests are handled
- Ensure credentials are properly configured

**3. Authentication Issues**
- Verify JWT secret is set correctly
- Check token expiration settings
- Ensure KV namespace is properly bound

**4. File Upload Errors**
- Verify R2 bucket is created and accessible
- Check file size limits (5MB default)
- Ensure proper file type validation

### Getting Help

**Check Logs:**
```bash
wrangler tail rawgle-backend --format=pretty
```

**Debug Mode:**
Set `DEBUG=true` in development environment for verbose logging.

**Health Endpoint:**
Use `/health` endpoint to verify service status and configuration.
