# ADR: Engineering Request Form Workflow Standardization

**Status:** Proposed

## Context
The company currently operates with fragmented workflows across processing and architecture divisions. The Engineering Request Form (ERF) process is inconsistent, leading to visibility issues and operational inefficiencies. The processing division has experienced significant losses ($500k/month for 3 months), highlighting the need for better project management visibility and workflow standardization. The current system lacks integration between ERF submission, shop production, and project management, creating distinct operational silos that impede effective oversight and problem prevention.

## Decision
Implement a unified Engineering Request Form workflow system that standardizes processes across all divisions (processing and architecture) while maintaining division-specific requirements. The system will provide real-time visibility into project status, integrate ERF submission with shop production scheduling, and establish clear handoff points between project management and production teams. Priority will be given to project management visibility features to enable proactive problem identification and resolution.
